SAWA · SPEC §6

Data Model & Reconciliation Queue

The tool keeps one record per person. Updates pour in from 3 tools that each name people differently. The Reconciliation Queue attaches each update to the right person — automatically when sure, via a review tray when not.

① Data comes in from 3 places — each identifies people differently
Dripify
knows people by → LinkedIn URL
connection sent · connected · message sent · replied (no text)
Calendly
knows people by → email, then company + name (booking form)
booked · cancelled
Landing page
knows people by → email (fallback: name+company)
new inbound signup
▼ ▼ ▼

🧠 Reconciliation Queue — "which of my people is this?"

Every incoming event runs down this ladder and stops at the first match:
1
LinkedIn URL Dripify events carry it
✓ auto-match
2
Email Calendly booking / landing form
✓ auto-match
3
Company + name from the Calendly booking form — e.g. only one "Anita" at Meridian
👤 confirm
4
No match new inbound signup → new lead · stray booking → tray
＋ new / tray
✓ Clean match → updates silently

The lead moves stage, the timeline gets an entry, suppression flips — no human needed. ~95% of events.

👤 Unsure → Review Tray

"This booking might be Anita at Meridian — confirm?" One tap. Near-empty on a normal day. Stops bad guesses from corrupting the numbers.
